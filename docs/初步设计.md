# 📄 MemeDetector 项目方案

## 1. 核心需求
* **采集：** 定时抓取 B 站高热分区的评论、弹幕。
* **发现：** 通过环比词频统计，识别出本周突然爆火的“疑似梗”。
* **分析：** 调用 LLM（DeepSeek-V3/Gemini）判定该词是否为亚文化梗，并自动化搜索溯源。
* **存储：** 结构化存储于轻量级搜索引擎，支持模糊匹配。

## 2. 最终技术栈 (Lean OSS Stack)

| 模块 | 选型 | 理由 |
| :--- | :--- | :--- |
| **语言** | **Python 3.12+** | 异步支持好，生态最强。 |
| **数据采集** | **bilibili-api-python** | 直接调用 B 站接口，简单粗暴。 |
| **词频分析** | **Jieba (极巴)** | 轻量级分词，支持自定义词典。 |
| **AI 逻辑** | **PydanticAI** | 基于函数调用，强类型约束，无冗余图逻辑。 |
| **LLM 推理** | **DeepSeek-V3** | 极高性价比，适合大规模语义分析。 |
| **存储/检索** | **Meilisearch** | Rust 编写，极轻量，比 OpenSearch 易用 10 倍。 |
| **部署管理** | **Docker Compose** | 一键拉起环境，保持开发生产一致。 |

## 3. 简化后的系统架构流转

### 第一步：采集与发现 (The Scout)
* **动作：** 脚本直接调用 `bilibili_api` 获取排行榜视频详情及评论。
* **处理：** 评论经由 `Jieba` 分词，结果存入临时 Redis 或本地 JSON。
* **逻辑：** 计算 $\text{Score} = \frac{\text{Current\_Freq}}{\text{Baseline\_Freq}}$。得分极高的词汇进入“待审名单”。

### 第二步：智能体分析 (The Researcher)
* **动作：** PydanticAI 接收到待审词。
* **流程：** 1.  工作流固定调用 **火山引擎联网搜索** 获取该词的背景，而不是把 B 站视频搜索暴露给 AI 作为工具。
    2.  AI 结合系统已执行好的搜索结果与评论上下文进行“语义建模”。
    3.  输出标准的 `MemeRecord` 对象（包含含义、出处、流行度）。

### 第三步：入库与展示 (The Archivist)
* **动作：** 格式化后的 JSON 数据直接 HTTP 推送到 Meilisearch。
* **检索：** 前端或 API 直接通过 Meilisearch 提供的 RESTful 接口进行极速查询。

## 5. 项目落地建议

1.  **分词调优：** 初始阶段，建议将“小鸡词典”或“萌娘百科”已有的梗词库导出，作为 `Jieba` 的初始词典，防止 AI 识别过多的普通词汇。
2.  **并发控制：** B 站对 API 调用有频率限制，直接调用时需加入 `asyncio.sleep` 或简单的代理池。
3.  **LLM 成本：** 推荐使用 DeepSeek-V3 处理批量的初步筛选，只有在需要视觉理解（看视频/表情包）时再调用 Gemini 1.5 Pro。
